  1. PBFT（Practical Byzantine Fault Tolerance）：一种经典的BFT算法，通过领导者选举、消息传递和投票过程来达成共识。在PBFT中，一个节点被选为领导者，负责提议和确认交易。

  2. Raft：一种简化版的BFT算法，用于在分布式系统中选举领导者，并通过心跳机制和投票流程来确保一致性。

  3. HotStuff：一种更现代的BFT算法，它通过优化投票流程，提高了共识速度和效率。

  4. Algorand：一种使用随机选择代表节点的BFT算法，以减少网络通信开销，提高共识效率。
