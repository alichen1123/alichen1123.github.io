什么是Layer 2？

1. 状态通道（State Channels）

2. 侧链（Sidechains）

侧链是独立的区块链，它们与主链通过特定的桥梁协议连接。侧链可以独立处理交易，然后将结果提交到主链进行确认。侧链如Polygon（原Matic Network）和Optimism就采用了这种模式，它们通过提供更低的gas费用和更快的交易确认时间，吸引了大量DApp开发者。

4. 优化的共识机制

这些解决方案通过改进共识算法，如权益证明（Proof-of-Stake, PoS）或分片（Sharding），来提高主链的处理能力。以太坊2.0的分片技术就是一种Layer 1的扩展方案，但同时也与Layer 2解决方案相辅相成，共同提升网络性能。

闪电网络：解决比特币的扩展性问题

闪电网络是由一系列节点和连接它们的通道构成的网络。每个通道都是两个节点之间建立的，允许他们在无需等待比特币网络确认的情况下进行多次交易。当交易发生时，它并不直接在比特币区块链上进行，而是通过闪电网络的节点进行路由，直到达到目的地。这样，闪电网络不仅提升了交易速度，还降低了交易费用。

路由算法：让交易快速找到最佳路径

在闪电网络中，路由算法负责找到从发送者到接收者的最佳路径，确保交易能以最快的速度、最低的费用完成。这个过程的关键在于，算法需要在数以万计的通道中找到一条既经济又可靠的路径。目前，闪电网络主要采用以下两种路由算法：

1. Flood+Propagate

这是闪电网络早期使用的路由算法，其工作原理是将交易广播到网络中的每个节点。每个节点收到交易后，会将交易信息传播给自己的邻居节点。这样，交易信息会像涟漪一样扩散，直到找到一条合适的路径。然而，这种算法的缺点是消息传播效率低，容易造成网络拥堵。

2. Flare (Flexible Logarithmic Routing)

为了克服Flood+Propagate的缺点，Flare算法应运而生。Flare是一种基于路径向量的路由算法，它利用了哈希图（Hash Graph）的数据结构。每个节点维护一个哈希图，记录了所有已知通道的哈希值。当交易需要路由时，节点会生成一个包含交易信息的哈希值，并通过哈希图找到与目标节点最接近的节点。这个过程重复进行，直到交易到达目标节点。Flare算法大大提高了路由效率，降低了消息传播的开销。
