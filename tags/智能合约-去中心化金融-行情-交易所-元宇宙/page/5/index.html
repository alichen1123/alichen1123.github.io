1. Optimism

Optimism是基于ZK-Rollup技术的Layer 2扩容方案，它利用零知识证明（Zero-Knowledge Proofs, ZKPs）来验证交易的有效性，从而将大部分交易处理过程移至Layer 2。Optimism的优点是交易速度快，gas费用低，且能够保持与以太坊主链的兼容性。然而，它的缺点在于ZKPs的复杂性可能导致技术门槛较高，且存在一定的隐私泄露风险。

2. Polygon (previously Matic)

Polygon，前身为Matic，采用的是Plasma技术，通过创建多个侧链来处理交易。侧链之间的数据同步由一个名为“信标链”的主链负责。Polygon的优点是易于扩展，能够处理大量交易，且gas费用相对较低。然而，Plasma的安全性依赖于主链，如果主链被攻击，侧链的安全性也会受到影响。

3. Arbitrum

Arbitrum采用Optimistic Rollup与零知识证明的结合，既保证了交易的快速处理，又降低了gas费用。它通过一个独立的执行层处理交易，然后将结果提交给以太坊主链验证。Arbitrum的优点是安全性高，但其交易确认时间相比其他方案可能稍长。

4. zkSync

zkSync是另一种基于ZK-Rollup的方案，它将交易数据压缩成零知识证明，然后在以太坊主链上验证。zkSync的优点是提供了与Optimism类似的性能提升，但其隐私保护更强，因为数据在验证之前已被压缩。

5. StarkEx

StarkEx利用STARK（ Scalable Transparent ARgument of Knowledge）技术，提供了一种安全、高效的扩容方案。STARKs允许在不泄露数据的情况下验证计算结果，使得StarkEx在保持高吞吐量的同时，具备了强大的隐私保护和抗欺诈能力。然而，STARK技术的复杂性使得开发和部署成本较高。
