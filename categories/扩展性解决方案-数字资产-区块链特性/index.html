状态通道是一种点对点的协议，允许交易双方在链下进行多次交易，只有在交易结束时才将最终状态广播到主链。这极大地减少了主链上的交易数量，从而提高了处理速度。

2. 侧链（Sidechains）

侧链是独立的区块链，可以与主链进行安全的资产互换。它们独立处理交易，只有在必要时才将数据同步到主链，从而降低了主链的负载。

3. 汇总（Rollups）

汇总技术将多个交易打包成一个“聚合交易”，在链下进行计算，然后将结果提交到主链。Rollups分为两种：Optimistic Rollup和ZK-Rollup，它们在保证安全性的同时，显著提高了交易效率。

  • 安全性：虽然Layer 2设计上要保证与主链的安全性一致，但实际操作中仍需面对新的安全挑战。

  • 互操作性：不同Layer 2方案之间的兼容性和互操作性是当前需要解决的问题。

中心化存储依赖于单一或少数几个中心节点，这些节点负责数据的存储、管理和分发。然而，这种模式存在显著的脆弱性：一旦中心节点遭受攻击或故障，可能导致大规模的数据丢失。此外，用户对数据的控制权有限，隐私保护也成为空谈。

为了确保数据的可靠性，去中心化存储网络通常会进行数据的冗余备份。即使部分节点失效，数据仍可通过其他节点恢复。同时，通过共识机制，网络可以自动检测并修复数据的不一致。

2.3 文件检索与激励机制

用户通过内容寻址（Content Addressable）来检索数据，而非传统的域名或IP地址。此外，网络中的节点通过提供存储和检索服务获得奖励，形成了一个激励机制，鼓励更多节点参与网络，共同维护数据的存储。
